# 📚 Flashcard App Development Plan

**Technologies Used**: HTML, CSS, PHP, **Firebase Realtime Database**

---

## Overview

This plan outlines a step-by-step approach to building your flashcard app, starting from setting up the foundational pages to implementing advanced features. The app will be structured using HTML, CSS for the frontend, PHP for backend logic, and **Firebase Realtime Database** as the database. No code is provided, but the plan details what needs to be built at each step.

---

## Stage 1: Setup and Basic Structure

### 1.1. Project Initialization

- **Create Project Directory Structure**
  - `/public`: Publicly accessible files (HTML, CSS, JavaScript)
  - `/app`: PHP scripts and application logic
  - `/config`: Configuration files (e.g., Firebase configuration)
  - `/assets`: Assets like images and fonts
- **Initialize Version Control**
  - Set up Git repository to manage code versions
- **Set Up Development Environment**
  - Install PHP and a web server (e.g., Apache)
  - Configure `php.ini` and other settings as needed
  - Install Composer for dependency management

### 1.2. Firebase Setup

- **Create Firebase Project**
  - Go to [Firebase Console](https://console.firebase.google.com/) and create a new project
- **Enable Realtime Database**
  - Set up Firebase Realtime Database in the project
- **Obtain Firebase Configuration**
  - Retrieve API keys and configuration details for the web app
- **Set Up PHP Firebase Library**
  - Install `kreait/firebase-php` library via Composer
    ```bash
    composer require kreait/firebase-php
    ```
- **Configure Firebase in PHP**
  - Set up Firebase credentials in `/config/firebase.php`

### 1.3. Basic Templates and Pages

- **Create Common Template Files**
  - `header.php`: Contains common `<head>` content and navigation menu
  - `footer.php`: Contains footer content and closing HTML tags
- **Create Home Page (`index.php`)**
  - Introduction to the app
  - Links to Sign Up and Log In pages
- **User Authentication Pages**
  - `signup.php`: Registration form for new users
  - `login.php`: Login form for existing users
  - `logout.php`: Script to log users out using Firebase Authentication

## Stage 2: User Authentication and Profile

- **Form Creation**
  - Fields: Username, Email, Password, Confirm Password

  - Check for empty fields, valid email format, password strength
- **Firebase Authentication**
  - Use Firebase Authentication API to create user accounts
- **Database Entry**
  - Store additional user profile data (e.g., preferences) in the `users` node in Firebase Realtime Database
- **Confirmation Message**
  - Inform the user of successful registration or errors

### 2.2. User Login (`login.php`)

- **Form Creation**
  - Fields: Email, Password
- **Authentication Process**
  - Authenticate user credentials using Firebase Authentication
- **Session Management**

- **Error Handling**
  - Provide feedback for incorrect credentials

## Project Overview
A comprehensive flashcard application focusing on spaced repetition learning, active recall, and gamification elements.

## Technical Stack
### Frontend
- HTML5
- CSS3 (with SASS preprocessor)
- JavaScript (ES6+)
- Bootstrap 5.3 for responsive design

### Backend
- PHP 8.1+
- Firebase SDK 9.0+
- Composer for dependency management

### Database
- Firebase Realtime Database
- Firebase Authentication
- Firebase Cloud Messaging

### Firebase Services Implementation

#### Authentication Setup
javascript
import { getAuth, createUserWithEmailAndPassword, signInWithEmailAndPassword } from "firebase/auth";

const auth = getAuth();

// User Registration
async function registerUser(email, password) {
try {
const userCredential = await createUserWithEmailAndPassword(auth, email, password);
return userCredential.user;
} catch (error) {
console.error("Registration error:", error);
throw error;
}
}

// User Login
async function loginUser(email, password) {
try {
const userCredential = await signInWithEmailAndPassword(auth, email, password);
return userCredential.user;
} catch (error) {
console.error("Login error:", error);
throw error;
}
}

#### Realtime Database Setup
javascript
import { getDatabase, ref, set, get } from "firebase/database";

const db = getDatabase();

// Write Data
async function writeUserData(userId, data) {
try {
await set(ref(db, 'users/' + userId), data);
return true;
} catch (error) {
console.error("Database write error:", error);
throw error;
}
}

// Read Data
async function readUserData(userId) {
try {
const snapshot = await get(ref(db, 'users/' + userId));
return snapshot.val();
} catch (error) {
console.error("Database read error:", error);
throw error;
}
}
